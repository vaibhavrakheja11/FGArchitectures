# Agentic AI Pipeline - Website

This category is reserved for website-facing AI architecture diagrams.

## Current Scope

Website AI currently focuses on the public assistant experience:

- Floating public chat widget.
- Visitor questions about games, services, careers, team, awards, and studio content.
- Profile-specific prompts for team member pages.
- Public context routing through the `flukegames` context.
- Direct chat responses through the public AI chat path.

## Planned Diagram

The future diagram should show:

1. Visitor question.
2. Public AI chat widget.
3. Website context selection.
4. Public studio data retrieval.
5. Public-safe response generation.
6. Follow-up questions and guided discovery.

## Technology Leadership Lens

The website AI layer should feel like guided discovery for visitors. It should answer public-safe questions, help visitors navigate content, and create a more interactive studio experience without exposing internal operations.
